Hapuque's Doces — Documentação Básica (detalhada)
=================================================

Visão geral
-----------
Hapuque's Doces é uma loja virtual estática (projeto TCC) construída com HTML/CSS/JavaScript puro. O site
permite navegar pelo cardápio, abrir um modal por produto, escolher quantidade, adicionar ao carrinho, e finalizar
enviando o resumo por WhatsApp. Há também páginas para Contato e Trabalhe Conosco com validação e notificações.

Tecnologias
-----------
- HTML5 (páginas em `html/`)
- CSS3 (`css/style.css`) — responsivo com media queries
- JavaScript (arquivos em `javascript/`) — manipulação do DOM, localStorage, eventos

Estrutura de arquivos (resumo)
------------------------------
- `index.html`            — Página inicial
- `html/`                 — Páginas internas:
  - `cardapio.html`      — Lista de produtos e modais de produto
  - `carrinho.html`      — Página do carrinho e finalização
  - `contato.html`       — Formulário de contato
  - `trabalhe.html`      — Formulário para envio de currículo
- `css/style.css`         — Estilos do layout, modais, toasts e responsividade
- `javascript/`           — Scripts principais:
  - `carrinho.js`        — Lógica do modal e operações do carrinho (adicionar, calcular subtotal)
  - `carrinho-render.js` — Renderização do carrinho, badge, eventos de UI
  - `menu.js`            — Comportamento do menu (hamburger, overlay)
  - `validacao-formularios.js` — Validações e notificações (contato/trabalhe)

Comportamento principal
-----------------------
- Carrinho
  - Persiste em `localStorage` (chave usada internamente, procurar por 'carrinho' no código).
  - Atualiza automaticamente ao alterar quantidades/excluir itens.
  - Em várias partes do código há um evento customizado para `carrinhoAtualizado` que sincroniza UI e badges.

- Modal de produto
  - Abre ao clicar em um produto no cardápio.
  - Permite escolher quantidade (controles + e -) e mostra subtotal do item antes de adicionar.

- Finalizar compra
  - Botão de finalizar constrói uma mensagem e abre o WhatsApp Web (wa.me) com o pedido.
  - Não há backend nem pagamento integrado.

- Formulários
  - `contato.html` e `trabalhe.html` possuem validação dos campos obrigatórios (Nome, Sobrenome, E-mail, Telefone).
  - Mensagem e arquivo (em `trabalhe.html`) são opcionais.
  - Após envio aparece um toast (notificação) de sucesso ou erro.

Como executar localmente (recomendado)
-------------------------------------
Opção A — VS Code Live Server (recomendado):
1. Abra a pasta do projeto no VS Code.
2. Instale/ative a extensão Live Server.
3. Clique em "Go Live" para abrir `index.html` localmente.

Opção B — Servidor simples via Python (se instalado):
1. No terminal, vá até a pasta do projeto e rode:

   python -m http.server 8000

2. Abra o navegador em `http://localhost:8000`.

2. Carrinho → conferir itens, alterar quantidades com +/- → ver subtotal e total atualizarem.

Configurações e pontos de atenção para desenvolvedores
-----------------------------------------------------
- LocalStorage: a chave e o formato do objeto do carrinho estão definidos em `carrinho.js` e em `carrinho-render.js`.
- Eventos: o código usa eventos customizados (ex.: `carrinhoAtualizado`) — se alterar a lógica, mantenha esses disparos
  para sincronizar as partes da UI.
- Notificações: estilos em `css/style.css` (.notificacao, .notificacao-sucesso, .notificacao-erro) e a função
  `mostrarNotificacao()` está em `validacao-formularios.js`.
- Internacionalização: há um widget do Google Translate incluído — revisar chaves e posição se necessário.

Limitações conhecidas
---------------------
- Sistema estático: não há persistência no servidor, nem autenticação.
- WhatsApp: depende do cliente web do WhatsApp; usuários sem WhatsApp Web ativo terão comportamento variado.
- Validação: validação é no front-end apenas; se for necessário, implementar backend para validações e envio real.

Notas do sistema de carrinho
---------------------------
Esta seção reúne a documentação específica do sistema de carrinho (originalmente em `javascript/README-CARRINHO.js`).

Funcionalidades principais:
- Modal do cardápio: abre ao clicar em um produto, exibe imagem, nome, preço e permite ajuste de quantidade com
  visualização de subtotal antes de adicionar ao carrinho.
- Gerenciamento do carrinho: salva produtos no localStorage, valida dados antes de adicionar, atualiza quantidades
  se o produto já existir e dispara um evento customizado `carrinhoAtualizado` para sincronizar a UI.
- Renderização do carrinho: lista itens com imagem, nome, preço unitário e subtotal; inclui botões + / - para
  alterar quantidade e um botão para remover item. Calcula total geral e mostra mensagem quando vazio.
- Finalização: recolhe dados do cliente (nome, telefone, endereço), gera um ID único (timestamp), monta a mensagem
  formatada com os itens e abre o WhatsApp Web (wa.me). O carrinho é limpo após envio.

Estrutura de funções (resumo):
- `carrinho.js`:
  - abrirModal(elementoDoce)
  - fecharModal()
  - atualizarSubtotal()
  - getCarrinho()
  - saveCarrinho(carrinho)
  - removerItem(id)
  - adicionarProdutoAoCarrinho()

- `carrinho-render.js`:
  - renderizarCarrinho()
  - mudarQuantidade(id, delta)
  - setupCarrinhoEventListeners()
  - handleCarrinhoClick(e)
  - limparCarrinho()
  - finalizarCompra()

Armazenamento:
- LocalStorage: chave utilizada `carrinhoHapuque`.
- Formato: JSON array com objetos contendo:
  {
    id: "produto-nome",
    nome: "Nome do Produto",
    preco: 10.50,
    imagem: "caminho/para/imagem.png",
    quantidade: 2
  }

Evento customizado:
- Ao salvar o carrinho, um evento `carrinhoAtualizado` é disparado para que outras partes da UI atualizem.

Como usar (trechos HTML de exemplo):
- Item do cardápio:
  <div class="card-3d doce-item">
    <img src="..." alt="..." />
    <h3>Nome do Doce</h3>
    <span class="card-price">R$ 0,00</span>
  </div>

- Modal de produto (exemplo mínimo):
  <div id="modal-produto" class="modal">
    <div class="modal-content">
      <span class="close-button" onclick="fecharModal()">&times;</span>
      <img id="modal-imagem" />
      <h3 id="modal-nome"></h3>
      <p id="modal-preco"></p>
      <input id="quantidade" type="number" value="1" min="1" />
      <span id="modal-subtotal"></span>
      <button id="btn-adicionar" onclick="adicionarProdutoAoCarrinho()">Adicionar ao Carrinho</button>
    </div>
  </div>

- Trechos do carrinho (exemplo mínimo):
  <div id="lista-carrinho" class="lista-carrinho-container"></div>
  <span id="contador-itens">0</span>
  <span id="valor-total-carrinho">R$ 0,00</span>
  <div id="carrinho-vazio">Carrinho vazio</div>
  <button onclick="limparCarrinho()">Limpar Carrinho</button>
  <button id="btn-finalizar" onclick="finalizarCompra()">Finalizar Compra</button>

Melhorias e boas práticas consideradas:
- Funções pequenas e reutilizáveis, validação de entrada, tratamento de erros e performance otimizada.
- Evitar duplicação de listeners e usar eventos customizados para sincronização entre módulos.

Fim das notas do sistema de carrinho.
